## 📖 第一部分：理論與 AI 學習法 (深度解析)

### 1. 什麼是提示工程 (Prompt Engineering)？
ChatGPT 的表現優劣，完全取決於你如何「發問」。一個好的提示詞 (Prompt) 應該包含：
* **角色設定**：例如「你是一位資深的資料庫架構師」。
* **任務描述**：明確說明你要寫 SQL、除錯，還是解釋概念。
* **限制條件**：例如「使用繁體中文」、「使用 pymysql 模組」、「加上註解」。
* **背景資料**：給予資料表的 Schema (結構)。

### 2. 使用 AI 學習資料庫理論
如果你對 ER 圖、正規化或事務處理 (ACID) 感到困惑，可以請 AI 用更白話的方式解釋。

> **💻 實戰練習：**
> 請對 AI 說：「請扮演一位幽默的教授。請用『金門特產店』的例子，向一位完全不懂資料庫的學生，解釋什麼是『資料庫正規化 (Normalization)』，以及為什麼要做它？」

### 3. 生成與最佳化 SQL 指令
當我們面對跨越多張表的複雜 JOIN 查詢時，可以先將資料表結構餵給 AI。

  ```sql
  -- AI 生成範例：
  -- 需求：查詢「金門縣」會員的購買總額。
  SELECT c.customer_name, SUM(p.price * o.qty) AS total_amount
  FROM customers c
  INNER JOIN order_details o ON c.customer_id = o.customer_id
  INNER JOIN products p ON o.product_id = p.product_id
  WHERE c.city = '金門縣'
  GROUP BY c.customer_name;
  ```

## 🤖 第二部分：AI 輔助程式開發 (Python 與 PHP 實戰)

這部分是本章的重頭戲。當我們完成 SQL 測試後，可以請 AI 直接把查詢封裝進程式碼中。

### 1. 產出 Python (PyMySQL) 程式碼
> [!TIP]
> **【給 AI 的精準提示詞】**
> 「請使用 Python 語言與 PEP8 風格寫一個程式，加上繁體中文註解。
> 使用 **pymysql** 模組連接 MySQL，查詢 `products` 表中價格大於 500 的商品。
> 主機：localhost，帳號：root，密碼：1234，資料庫：kinmen_shop。」

### 2. 產出 PHP (mysqli) 程式碼
> [!TIP]
> **【給 AI 的精準提示詞】**
> 「請將上述功能改用 **PHP** 語言撰寫。
> 使用 `ext/mysqli` 擴充程式，並以表格 (HTML Table) 顯示商品名稱與價格。
> 請記得處理 UTF-8 編碼與連線錯誤。」

> **🧠 思維訓練：為什麼要讓 AI 產出兩種語言？**
> 在現代開發中，後端資料分析常用 Python，而前端展示常用 PHP。學會利用 AI 進行「語言間的邏輯轉換」，能讓你更靈活地選擇適合的工具。

## 💻 第三部分：AI 除錯與效能調校 (DBA 進階技巧)

當程式報錯或查詢太慢時，ChatGPT 是最好的初階除錯工具。

### 1. 錯誤訊息解析
當你在 Workbench 看到紅色的 `Error Code: 1064` 時，別慌張！
* **做法**：直接複製整個錯誤代碼與你的 SQL 語句貼給 AI。
* **提示詞**：「我的 SQL 出現 1064 錯誤，請幫我檢查語法哪裡出錯了？」

### 2. SQL 效能優化建議
* **提示詞**：「這是我目前的 SQL 語句：`(貼上語法)`。這張表有 100 萬筆資料，請幫我分析這段語法是否會造成全表掃描，並建議我應該在哪些欄位建立索引 (Index)？」

## ❓ 第四部分：常見問題與排解 (FAQ)

1. **Q: ChatGPT 給我的程式碼執行起來有錯誤，怎麼辦？**
   * A: AI 有時會產生「幻覺」或是使用過時的函式庫。
     * **對策一**：把錯誤訊息回傳給 AI，讓它自我修正。
     * **對策二**：對照課本或官方文件 (如 PyMySQL 官方文件) 進行最後檢查。
2. **Q: 在公司或學校使用 AI 有資安疑慮嗎？**
   * A: **絕對有！** 嚴禁將公司的真實資料、真實帳號密碼或具備隱私的個資貼給 AI。請使用假資料 (Dummy Data) 進行測試。
3. **Q: 既然 AI 這麼會寫 SQL，我還需要學 SQL 語法嗎？**
   * A: **非常需要！** AI 就像計算機，而 SQL 就是數學。如果你不懂加減乘除，你連計算機按出來的答案是對是錯都不知道。掌握基礎語法，你才能驗證 AI 的正確性。

## 📝 第五部分：實戰演練與魔王挑戰

### 🟢 基礎題 (AI 對談練習)
1. **理論白話文**：請對 AI 說：「請用『夜市攤位』的例子解釋資料庫的 Primary Key 和 Foreign Key 的關係。」
2. **SQL 生成**：請給 AI 你的 `products` 表結構，並請它寫出一段 SQL，找出「所有類別為『伴手禮』且庫存大於 100 的商品」。

### 🟡 進階題 (程式碼區塊置換)
3. **功能改寫**：假設你已經有一支能「新增商品」的 Python 程式。請將該程式貼給 AI，並要求：「請幫我修改這支程式，在寫入資料庫前，先用 Python 判斷商品價格是否為負數，如果是負數則不准寫入並報錯。」
4. **跨語言翻譯**：請 AI 將你在第十六章寫的「會員註冊 PHP 程式」，完整翻譯成「Python Streamlit 版本」。

### 😈 魔王挑戰題 (AI 驅動的全自動報表系統)
5. **【AI 結對程式設計挑戰】**
   * **情境**：老闆希望有一個網頁，可以自動統計「每位會員的總消費金額」，並畫成「圓餅圖」。
   * **挑戰任務**：
     - 步驟 A：請 AI 幫你寫出三表 JOIN (customers, order_details, products) 並進行 GROUP BY 的 SQL 語法。
     - 步驟 B：請 AI 寫一段 Streamlit 程式，執行上述 SQL，並利用 `st.plotly_chart` (這是一個進階圖表模組，請 AI 教你怎麼用) 畫出圓餅圖。
     - 步驟 C：如果 AI 生成的程式碼缺少 `conn.commit()` 或是連線資訊錯誤，請你根據本學期所學，手動將其修復！
   * **過關標準**：成功在網頁上看到由 AI 輔助產生、但由你親自除錯完成的動態圖表！
